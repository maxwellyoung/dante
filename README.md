# Dante

Experimental game and engine work around a small playable world, Lua gameplay systems, and a native `ev_engine` runtime.

This repository is public as a technical artifact rather than a polished product. It shows low-level gameplay exploration, entity/system design, camera and encounter logic, and a C/C++ engine direction alongside Lua iteration.

## What it demonstrates

- Lua gameplay systems for enemies, encounters, collectibles, effects, and camera behavior.
- A native engine workspace under `ev_engine/`.
- Iterative game design notes and implementation plans.
- Comfort moving between scripting, runtime architecture, and interaction design.

## Repository map

- `*.lua`: gameplay systems and prototype logic.
- `ev_engine/`: native runtime and engine notes.
- `BLENDER_PIPELINE.md`: asset pipeline planning.
- `GAMEPLAY_REVISION.md`: gameplay direction and revision notes.
- `MASTER_PLAN.md`: broader project planning.

## Status

Prototype. Useful for reading implementation direction and engine/gameplay thinking; not packaged as an end-user release.
